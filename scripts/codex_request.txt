The current EC2 failure is no longer primarily about fee accounting, broker balance visibility, stale halt clearing, or open-position resume policy. The remaining failure is in **position authority projection/rebuild semantics**: `full_projection_rebuild` deletes fragmented `open_position_lots` rows, but replay recreates the same historical dust rows, leaving the materialized projection divergent from current broker/portfolio truth while still surfacing an unsafe `--apply --yes` recommendation.

## Mission

Do **not** solve this by flattening the position, forcing resume, manually editing the DB, repeatedly applying repair commands, suppressing blockers, or weakening safety gates.

The goal is to make `position_authority` a broker/portfolio-anchored, provenance-aware projection system.

Core principle:

```text
open_position_lots is a rebuildable materialized projection,
not canonical truth.
```

Canonical truth should be:

```text
fills
trades
finalized fee events
external cash/position adjustments
manual repair events
current broker account snapshot as evidence
```

Derived projections should be:

```text
open_position_lots
trade_lifecycles
normalized exposure
dust tracking
```

The current failure shows that `full_projection_rebuild` is not terminally anchored to current broker/portfolio state. It replays historical fills and recreates invalid fragmented dust rows. The patch must fix the rebuild semantics, final safety gates, and operator output.

## Required design changes

### 1. Make full projection rebuild terminally broker/portfolio anchored

A successful `full_projection_rebuild` must satisfy:

```text
post_rebuild_projected_total_qty == portfolio_qty == broker_qty
```

within the repository’s quantity tolerance.

For this incident, successful post-state must be:

```text
post_rebuild_projected_total_qty=0.00039985
portfolio_qty=0.00039985
broker_qty=0.00039985
projection_converged=true
```

If replay produces:

```text
projected_total_qty=0.00218797
portfolio_qty=0.00039985
projected_qty_excess=0.00178812
projection_converged=false
```

then final result must be:

```text
final_safe_to_apply=false
recommended_command absent
operator_next_action=review_rebuild_replay
```

Do not allow apply to succeed or be recommended if final projection is still divergent.

### 2. Add broker-anchor collapse / trim / re-anchor step after historical replay

Historical replay may be useful as an intermediate diagnostic step, but full rebuild must publish a materialized projection that matches current broker/portfolio truth.

Implement one of these behaviors:

```text
A. Replay historical fills/trades, then collapse/trim/re-anchor projection rows to broker_qty/portfolio_qty before publish.

B. If collapse/trim/re-anchor cannot be done safely because required evidence is missing or contradictory, fail closed:
   final_safe_to_apply=false
   recommended_command absent
   operator_next_action=review_rebuild_replay
```

Do not publish or recommend a projection that keeps resurrected historical dust rows beyond broker/portfolio anchor.

### 3. Separate pre-gate safety from final apply safety

Introduce distinct fields in the internal result and operator output:

```text
pre_gate_passed
replay_projection_converged
post_publish_projection_converged
post_rebuild_projected_total_qty
portfolio_qty
broker_qty
final_safe_to_apply
recommended_command
operator_next_action
```

Rules:

```text
safe_to_apply must either mean final_safe_to_apply, or be renamed so it cannot be mistaken.
recommended_command with --apply --yes may appear only when final_safe_to_apply=true.
```

If only pre-gates pass but replay/post-publish does not converge:

```text
pre_gate_passed=true
replay_projection_converged=false
post_publish_projection_converged=false
final_safe_to_apply=false
recommended_command absent
operator_next_action=review_rebuild_replay
```

### 4. Treat `open_position_lots` as invalid projection when it diverges from broker/portfolio

When:

```text
projected_total_qty != portfolio_qty
```

do not let `open_position_lots` define true dust/exposure state. It is invalid projection evidence.

Expected policy:

```text
canonical_state=PROJECTION_INVALID or equivalent
run_loop_allowed=false
entry_allowed=false
exit_allowed=false
operator_action=repair/diagnose
```

Do not present the position as a normal `dust_only` state when projection is divergent.

### 5. Preserve provenance-aware invariants

Keep and strengthen the provenance model:

```text
FillNativeLot:
  source=ledger/accounted_buy_evidence
  invariant=fill qty + lifecycle residual

PortfolioAnchorProjectionLot:
  source=full_projection_rebuild_portfolio_anchor
  invariant=broker/portfolio convergence

ManualRepairProjectionLot:
  source=manual/portfolio repair
  invariant=repair event + broker/portfolio convergence
```

Rules:

```text
target_lot_qty_fill_mismatch may only apply to FillNativeLot.
PortfolioAnchorProjectionLot must be checked by broker/portfolio convergence, not original BUY fill qty.
ManualRepairProjectionLot must require repair evidence plus broker/portfolio convergence.
```

### 6. Make repair-plan output stage-aware

Operator output must distinguish:

```text
pre-assessment
replay result
publish candidate
post-publish assessment
final recommendation
```

Add or expose fields equivalent to:

```text

post_publish_projected_total_qty
portfolio_qty

semantic_contract_check_applicable
semantic_contract_check_skipped_reason
pre_gate_passed
final_safe_to_apply
recommended_command
rollback_path
why_safe
why_unsafe
```

The operator must never see an unqualified `safe_to_apply=true` next to final evidence showing `projection_converged=false`.
