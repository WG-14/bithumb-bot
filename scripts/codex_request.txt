  -> residual submit plan is silently lost

Your job is to verify this directly in the repo, fix it if confirmed, and add regression tests that prove the default `build_signal_execution_service()` path carries `execution_submit_plan` all the way to the live submit path without silently dropping it.

residual_sell_candidate:
  authority = residual inventory policy
  source = residual inventory model
  used only through explicit residual policy + final pre-submit proof

## Required design changes

### 1. Preserve execution_submit_plan through the default live wrapper

`execution_submit_plan` must be a supported keyword on:

```text
execution_service.live_execute_signal(...)
broker.live.live_execute_signal(...)
```

and must reach the live submit path.

Do not rely on custom test-only executors to prove this. The default service path must work.

### 2. Residual enabled path must fail closed if plan-aware executor support is missing

In `LiveSignalExecutionService.execute()`:

```text
If request.signal == SELL
and residual_submit_plan.source == residual_inventory
and residual_submit_plan.submit_expected == true
and RESIDUAL_LIVE_SELL_MODE == enabled
then execution_submit_plan is required.

If executor rejects execution_submit_plan:
  do not call executor again without plan
  return explicit blocked result or raise a controlled error
  no broker order
```

Choose the repo-consistent failure style. The important point is no silent plan loss.

A test must assert this behavior.

### 3. Broker live path must consume residual execution_submit_plan through standard pipeline

`broker.live.live_execute_signal()` must recognize an `execution_submit_plan` with:

```text
source=residual_inventory
authority=residual_inventory_policy
side=SELL
intent_type=residual_close
```

and convert it into the existing standard live submit pipeline request.

It must preserve:

```text
side=SELL
qty=residual_submit_plan.qty
submit_qty_source=residual_inventory_policy
intent_type=residual_close
strategy_context=residual_inventory_policy
decision_reason_code=CLOSE_RESIDUAL_CANDIDATE
```

It must not:

```text
- treat residual qty as canonical strategy sellable_executable_qty
- mutate lot-native strategy SELL authority
- bypass standard submit planning / order state / dedup / recovery machinery
```

### 4. BUY delta live path must be proven

If not already wired, wire `RESIDUAL_BUY_SIZING_MODE=delta` into the actual live BUY sizing call.

Actual live BUY sizing must use residual-aware effective exposure under delta mode. Telemetry-only mode must not change actual sizing.

Expected policy:

```text
RESIDUAL_BUY_SIZING_MODE=telemetry:
  compute/report residual delta
  existing BUY sizing remains unchanged

RESIDUAL_BUY_SIZING_MODE=delta:
  existing exposure + tracked residual exposure reduce buy budget
```

### 5. Keep accounting proof fail-closed

Do not reintroduce:

```python
payload["accounting_projection_ok"] = bool(readiness_payload.get("projection_converged"))
```

Missing accounting projection proof must fail closed.
