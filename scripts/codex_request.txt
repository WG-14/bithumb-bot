## Mission

Do **not** solve this by flattening the position, forcing resume, manually editing the DB, repeatedly applying repair commands, or hiding blockers.

The goal is to make `position_authority` a provenance-aware projection system:

```text
canonical truth =
  fills
  trades
  finalized fee events
  external cash/position adjustments
  manual repair events
  current broker account snapshot as evidence

derived materialized projections =
  open_position_lots
  trade_lifecycles
  normalized exposure
  dust tracking
```

`open_position_lots` must not act as untyped canonical truth. It must be a rebuildable projection whose invariants depend on provenance.

The key principle:

```text
fill-native lot evidence
≠ broker/portfolio-anchored projection evidence
≠ manual portfolio repair evidence
```

## Required design changes

### 1. Make full projection rebuild broker/portfolio-anchored at terminal state

If the command is called `full_projection_rebuild`, its final success invariant must be:

```text
post_rebuild_projected_total_qty == portfolio_qty == broker_qty
```

within the repository’s quantity tolerance.

This must be true after publish/write, not only in a preview sub-object.

If historical replay creates old dust rows that exceed current portfolio/broker quantity, the rebuild must either:

```text
collapse / trim / re-anchor projection rows to broker/portfolio quantity
```

or fail closed with:

```text
safe_to_apply=false
operator_next_action=review_rebuild_replay
```

Do not allow a successful apply that leaves:

```text
projected_total_qty != portfolio_qty
```

### 2. Make projection row invariants provenance-aware

Introduce or strengthen typed internal concepts equivalent to:

```text
FillNativeLot:
  source_mode=ledger or equivalent
  invariant=original fill qty + lifecycle residual

PortfolioAnchorProjectionLot:
  source_mode=full_projection_rebuild_portfolio_anchor
  invariant=current broker/portfolio projection convergence

ManualPortfolioRepairLot:
  source_mode=portfolio_anchored_repair
  invariant=repair event + current broker/portfolio convergence
```

Rules:

```text
target_lot_qty_fill_mismatch may only be emitted for FillNativeLot.
```

For `PortfolioAnchorProjectionLot`, skip fill-native invariant and report:

```text
semantic_contract_check_applicable=false
semantic_contract_check_skipped_reason=portfolio_anchor_projection_lot
```

For `ManualPortfolioRepairLot`, skip fill-native invariant only if matching repair/publication evidence is present and current broker/portfolio convergence holds.

### 3. Ensure target provenance is reloaded after rebuild/publish

If rebuild writes new `open_position_lots` rows with source mode:

```text
full_projection_rebuild_portfolio_anchor
```

then any post-assessment must reload target rows from the DB and re-run provenance classification.

Do not reuse stale pre-rebuild classification.

Expected:

```text
post_publish_target_lot_provenance.kind=portfolio_anchor_projection_lot
post_publish_target_lot_provenance.fill_qty_invariant_applies=false
```

### 4. Fix multi-lot target residual vs portfolio total comparison

Where `portfolio_target_qty_mismatch` is generated, ensure the check uses:

```text
portfolio_target_remainder_qty = max(0, portfolio_qty - other_active_qty)
```

or equivalent.

For a multi-lot case:

```text
portfolio_qty=0.00049980
other_active_qty=0.00039982
target_residual_qty=0.00009998
```

the correct result is:

```text
portfolio_target_remainder_qty=0.00009998
no portfolio_target_qty_mismatch
```

Do not accept `other_active_qty` blindly. It must itself be part of a converged projection or supported by broker/portfolio/replay evidence.

### 5. Enforce final repair-plan/apply consistency

A repair command may show intermediate preview details, but final recommendation must depend on final post-state.

Required final gates:

```text
broker_portfolio_converged=true
accounting_projection_ok=true
post_rebuild_projection_converged=true
post_rebuild_projected_total_qty == portfolio_qty
active fee issue=0
unresolved local order=0
remote open order=0
submit_unknown=0
semantic contract passed
  OR semantic contract intentionally skipped due to recognized projection provenance
```

If any final gate fails:

```text
safe_to_apply=false
recommended_command absent
operator_next_action=review_position_authority_evidence or review_rebuild_replay
```

