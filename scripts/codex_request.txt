## Core problem statement

The repository should be able to execute this validation lifecycle as code:

```text
hypothesis
→ data collection / dataset snapshot
→ market regime or segment definition
→ candidate parameter-space definition
→ first-pass backtest
→ parameter exploration
→ overfitting checks
→ out-of-sample validation
→ walk-forward validation
→ paper / dry-run validation
→ small live candidate readiness
→ runtime env/profile promotion
→ operational log revalidation
```

The key distinction must be preserved:

```text
Research-stage env/config values = experimental candidate variables
Runtime env values = verified output of the validation process
```

Do not collapse those two concepts into “just edit `.env` until it works”.
