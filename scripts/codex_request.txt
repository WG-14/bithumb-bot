## Desired end state

The bot should support this operator-level behavior:

```text
The bot buys.
The bot sells.
The exchange may leave leftover BTC due to fees, fills, rounding, min qty, or min notional.
The bot may stop and restart.
After restart, the bot reads broker-verified current BTC, validates readiness/recovery safety gates, compares current position to persisted target position, and converges by delta.
If target=0 and leftover BTC is executable, it sells it.
If target=0 and leftover BTC is below exchange minimum, it records/no-ops as true_dust.
The state machine must not get stuck because leftover BTC is not a strategy lot.
```

## Required implementation details

### LiveSignalExecutionService target_delta branch

Implement an explicit branch near the top of `LiveSignalExecutionService.execute()`:

```python
if _execution_engine() == "target_delta":
    execution_decision = ...
    target_plan = execution_decision.get("target_submit_plan")
    if not isinstance(target_plan, dict):
        return None
    if target_plan.get("source") != "target_delta":
        return None
    if target_plan.get("block_reason") != "none":
        return None
    if not target_plan.get("submit_expected"):
        return None
    if target_plan.get("side") not in {"BUY", "SELL"}:
        return None
    if float(target_plan.get("qty") or 0.0) <= 0.0:
        return None

    return self.executor(
        self.broker,
        target_plan["side"],
        request.ts,
        request.market_price,
        strategy_name=request.strategy_name,
        decision_id=request.decision_id,
        decision_reason=request.decision_reason,
        exit_rule_name=request.exit_rule_name,
        execution_submit_plan=target_plan,
    )
```

Use the repo’s actual helper names and style. Do not copy this blindly if the surrounding API differs.
