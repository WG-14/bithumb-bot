## Current High-Priority Hypotheses

Treat these as hypotheses, not conclusions. Verify or falsify each.

### Hypothesis A: Partial-close residual convergence uses an overly strict quantity identity tolerance

The strongest current hypothesis is that `position_authority_state.py` correctly computes target-entry lifecycle matched quantity:

```text
target_lifecycle_matched_qty = 0.0004
effective_closed_qty = 0.0004
expected_residual_qty = 0.00019998
```

but then compares the materialized residual `0.00019986` to `0.00019998` using `_EPS = 1e-12`, which is appropriate for strict identity checks but not for exchange/residual/lot-authority quantity convergence. This causes an otherwise converged partial-close dust residual to be misclassified as `needs_correction=True`.

Verify by inspecting:

```text
_EPS usage in position_authority_state.py
_partial_close_residual_state_converged()
partial_close_residual_candidate classification
target_total_qty / target_dust_qty / expected_residual_qty comparisons
blocker construction for target_lot_qty_fill_mismatch
```

Expected confirmation signal:

A regression fixture with:

```text
BUY B fill_qty = 0.00059998
SELL matched to B = 0.0004
expected_residual = 0.00019998
materialized dust row / portfolio / broker = 0.00019986
delta = 0.00000012
```

currently falls into:

```text
partial_close_residual_candidate=false
residual_state_converged=false
needs_correction=true
```

when it should be either:

```text
partial_close_residual_candidate=true
residual_state_converged=true
needs_correction=false
recovery_stage=RESUME_READY
resume_blockers=()
run_loop_allowed=true

```

or, if the current state is not yet converged by design:

```text

safe_to_normalize_residual=true
preview.repair_mode=residual_normalization
preview.safe_to_apply=true
```

Given the production state already has a single `dust_tracking` row, broker/portfolio/projection convergence, and a projection publication, the preferred classification is likely “already converged partial-close residual state,” not a repair-required state.

Reject or weaken this hypothesis only if the production-like fixture reveals missing lifecycle evidence, missing projection publication, broker/portfolio divergence, open exposure, or another real safety issue.

### Hypothesis B: Temporal SELL-after-target totals and target-entry lifecycle attribution are not fully separated

The current code appears to have helper logic like `_load_target_lifecycle_matched_qty()` and `effective_closed_qty = target_lifecycle_matched_qty if accepted else sell_after_qty`.

Verify this directly.

If lifecycle attribution is already correctly used for `expected_residual_qty`, do not claim this is the primary root cause. Instead, identify any remaining places where:


Authority decisions should use target-entry lifecycle attribution when available. Total later SELL quantity should remain visible for diagnostics, but it should not by itself block a partial-close residual state that has valid target lifecycle evidence and converged projections.

## Preferred Design Direction

The immediate code change should be minimal but principled.

### 1. Promote quantity-contract-aware residual comparisons

Introduce a dedicated residual authority quantity tolerance or comparison policy. Do not reuse `_EPS = 1e-12` for every exchange/residual/lot-authority comparison.

The tolerance must be bounded and contextual. It should be based on available contract data such as:

```text
internal_lot_size
qty_step
max_qty_decimals
exchange/broker quantity precision
broker/local reconciliation tolerance
dust boundary
```

A starting implementation may be local to `position_authority_state.py`, but structure it so it can later become a centralized quantity comparison policy.

A possible shape:

```python
def _residual_qty_tolerance(
    *,
    internal_lot_size: float,
    qty_step: float = 0.0,
    max_qty_decimals: int | None = None,
) -> float:
    decimal_unit = 0.0
    if max_qty_decimals is not None and max_qty_decimals > 0:
        decimal_unit = 10 ** (-int(max_qty_decimals))

    return max(
        _EPS,
        decimal_unit * 20,
        abs(float(qty_step or 0.0)) * 0.002,
        abs(float(internal_lot_size or 0.0)) * 0.001,
        2e-7,
    )
```

Do not blindly copy this if the repository already has a better quantity policy. Verify existing helpers first. If you use a constant like `2e-7`, document why it is bounded, why it covers the observed `1.2e-7 BTC` residual delta, and why it cannot accidentally grant executable authority for a full internal lot.

### 2. Apply the residual tolerance only to residual convergence, not to all authority checks

Use the residual tolerance for comparisons like:

```text
abs(target_total_qty - expected_residual_qty)
abs(target_dust_qty - expected_residual_qty)
abs(lifecycle_matched_qty - expected_closed_qty)
```

when and only when the context is a partial-close residual candidate or convergence check with lifecycle evidence.

Do not use this tolerance to authorize executable exposure. Do not use it to hide missing lifecycle rows, missing broker evidence, open exposure conflicts, or projection divergence.

### 3. Keep lifecycle matched quantity as first-class authority evidence

Ensure partial-close residual logic uses:

```text
target_lifecycle_matched_qty
lifecycle_matched_qty_accepted
effective_closed_qty
expected_residual_qty
```

for target-entry residual reasoning.

Keep `sell_after_target_buy_qty` visible as a diagnostic field, but do not let total later SELL qty override valid target lifecycle attribution.
