## Primary problem statement

The current working hypothesis is:

`ResearchMetrics` and the research acceptance/promotion flow already include important fields such as total return, MDD, profit factor, win rate, trade count, max consecutive losses, fees, slippage, and stability. However, promotion-grade research metrics do not clearly include CAGR, average holding time, exposure/time-in-market, global expectancy per trade, open-position-at-end state, and realized/unrealized separation. More importantly, research metrics, regime metrics, runtime performance metrics, acceptance gates, candidate profiles, and promotion artifacts appear to use related but not fully unified metric definitions.

The goal is not a symptom patch such as “add CAGR as one extra field.” The goal is to introduce a non-breaking metrics contract and shared calculation path that makes these metrics first-class, deterministic, observable, testable, and safe for future promotion/readiness gates.

### 2. Define formulas explicitly in code and docs

Use deterministic, documented formulas. Suggested definitions:

* `total_return_pct`: same meaning as current `return_pct`.
* `cagr_pct`: annualized return over elapsed calendar time; return `None` if elapsed time is zero or invalid.
* `realized_return_pct`: realized closed-trade net PnL divided by starting cash.
* `unrealized_pnl_end`: final equity minus cash minus realized basis as appropriate; if exact basis is not safely available, compute from final mark and open position cost basis and document the formula.
* `open_position_at_end`: true if the simulation ends with nonzero asset quantity or an open position interval.
* `expectancy_per_trade_krw`: mean net PnL over closed trades.
* `expectancy_per_trade_pct`: mean closed-trade return percentage using entry notional/cost basis if available; otherwise `None` with a limitation reason.
* `payoff_ratio`: `avg_win / abs(avg_loss)`, `None` if no losses or no wins unless the repo already has a preferred convention.
* `exposure_time_pct`: total time with active position divided by elapsed evaluation window.
* `avg/median/max_holding_time_ms`: computed over closed position intervals only; open interval should be separately represented and should not silently contaminate closed holding-time stats.
* `fee_drag_ratio` and `slippage_drag_ratio`: define against a stable denominator; prefer absolute gross PnL or total traded notional if available. Choose one, document it, and test it.
* `execution_count`: execution attempts, not closed trades.
* `closed_trade_count`: closed SELL-side realized trades/position closures.

Align these definitions with `strategy_performance.py` and `market_regime/metrics.py` where possible. If formulas diverge, either unify them or explicitly document why.
