## Required design changes

### 1. Add explicit residual live sell policy mode

Introduce a config setting with conservative default:

```text
RESIDUAL_LIVE_SELL_MODE=telemetry
```

Supported values:

```text
telemetry:
  Build residual sell candidate and pre-submit proof.
  Never submit a live residual SELL.
  final_action may be CLOSE_RESIDUAL_CANDIDATE.
  submit_expected=false.
  block_reason should explicitly say residual live sell mode is telemetry-only.

dry_run:
  Build a residual submit plan and all proof artifacts.
  Do not send a live broker order.
  Telemetry must show that a real order would have been submitted if enabled.

enabled:
  If and only if:
    residual candidate exists
    residual policy allows sell
    final pre-submit proof passes
    live execution is armed
    LIVE_DRY_RUN=false
    existing risk/order/recovery gates pass

```

Default must remain `telemetry`.

Do not rely on a code comment like “telemetry-only in this patch.” This must be a first-class config/policy visible in logs/tests/operator reports.
