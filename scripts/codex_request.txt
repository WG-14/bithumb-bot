## Required design changes

### 1. Change stale halt clear predicate from reason-whitelist-centered to active-blocker-absence-centered

Do not rely primarily on `last_reconcile_reason_code` whitelist. Reason codes can remain useful diagnostic context, but stale halt clear should be governed by current active evidence.

Desired predicate shape:

```text
halt_reason_code == RISK_STATE_MISMATCH
last_reconcile_status == ok
startup_gate_reason is None
accounting_projection_ok == true
broker_portfolio_converged == true
lot_projection_converged == true
broker_qty_known == true
broker snapshot stale == false
unresolved_fee_state == false
unresolved_open_order_count == 0
recovery_required_count == 0
remote_open_order_count == 0
submit_unknown_count == 0
fee_gap_resume_blocking == false
external_position_repair_needed == false
position_authority_repair_needed == false
risk_loss_breach == false
kill_switch == false
panic_stop == false
```

Allow stale halt clear even when:

```text
last_reconcile_reason_code=FEE_GAP_RECOVERY_REQUIRED
```

only if the fee-gap incident is historical/repaired and non-blocking:

```text
fee_gap_active_issue=false
fee_gap_resume_blocking=false
matching fee-gap accounting repair already recorded
```

Continue to block if:

```text
fee_gap_active_issue=true
fee_gap_resume_blocking=true
current broker/portfolio mismatch exists
active fee pending exists
projection drift exists
unresolved order exists
submit_unknown exists
broker snapshot stale/missing
kill switch/loss limit/panic stop active
source conflict active
```

Do **not** clear true risk halts. This change is only for stale `RISK_STATE_MISMATCH` after verified convergence.

### 2. Add clear failure observability

When `RISK_STATE_MISMATCH` remains unresolved, operator output must explain why.

Add fields to `health`, `recovery-report`, `restart-checklist`, or a shared diagnostic object:

For the current EC2 case, expected diagnostic before the fix would be something like:

```text
stale_halt_clear_candidate=true
current_evidence_converged=true
fee_gap_resume_blocking=false
stale_halt_clear_blockers=last_reconcile_reason_code_not_whitelisted:FEE_GAP_RECOVERY_REQUIRED

After the fix, this should become:

```

This is not `resume --force`. It is normal resume based on current verified evidence.

### 4. Redefine `HALT_RISK_OPEN_POSITION`

`HALT_RISK_OPEN_POSITION` must not mean “there is open exposure.”

It should mean:

```text
open exposure exists
AND current unresolved risk/evidence blocker exists
```

Examples where it may be valid:

```text
open exposure + current broker/portfolio mismatch
open exposure + stale/missing broker snapshot
open exposure + unresolved local or remote order
open exposure + submit_unknown
open exposure + active fee pending
open exposure + projection drift
open exposure + kill switch/loss limit/panic stop
```

Examples where it must not be emitted:

```text
open exposure + broker/portfolio/replay/projection converged + active risk none
```

For clean canonical open exposure, guidance should say:

```text
Canonical open exposure is present and accounted.
Resume in position-management mode is allowed.
New entries are blocked or strategy-controlled while the existing position is open.
Emergency flatten is available but not the primary recommendation.
```

### 5. Keep fail-closed behavior for unsafe states

Do not weaken safety.

Continue to block run/resume if any active blocker exists:

```text
broker/portfolio mismatch
active fee pending
unresolved local order
stray remote open order
submit_unknown
projection drift
broker snapshot stale/missing
risk loss limit breach
kill switch
panic stop
source conflict
```
