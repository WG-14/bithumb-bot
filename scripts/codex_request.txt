Hypothesis 1:
The repo already has public top-of-book fetch helpers, but no persisted `orderbook_top_snapshots` research data table.

Verification:
Inspect `marketdata.py`, `public_api_orderbook.py`, and `db_core.py`. Confirm whether a table exists and whether any command stores validated best bid/ask snapshots.

If not implemented, add it.

Required schema:

```sql
CREATE TABLE IF NOT EXISTS orderbook_top_snapshots (
    ts INTEGER NOT NULL,
    pair TEXT NOT NULL,
    bid_price REAL NOT NULL,
    ask_price REAL NOT NULL,
    spread_bps REAL NOT NULL,
    source TEXT NOT NULL,
    observed_at_epoch_sec REAL,
    PRIMARY KEY (ts, pair, source)
);
```

Add an index suitable for nearest-snapshot lookup:

```sql
CREATE INDEX IF NOT EXISTS idx_orderbook_top_pair_ts
ON orderbook_top_snapshots(pair, ts);
```

Validation requirements:

* bid and ask must be finite positive numbers.
* bid must be `<= ask`.
* spread_bps must be finite and non-negative.
* pair/market normalization must follow the repo’s existing market normalization conventions.
* `source` must be explicit, for example `bithumb_public_v1_orderbook`.
* Do not store fabricated quotes.
* Do not claim this table contains full depth.

Hypothesis 2:
Research backtests currently have `ExecutionRequest.best_bid`, `best_ask`, and `spread_bps` fields available, but the research path still leaves them empty because it only uses candle close as the execution reference.

Verification:

If not implemented, add quote-aware research join as an optional dataset layer.

Design requirements:

* Do not break existing candle-only SMA research.
* Add optional top-of-book support for research datasets.
* Join signal candle timestamp to the nearest orderbook top snapshot within a configurable tolerance.
* Default tolerance should be conservative and explicit, for example 3000 ms, unless a better existing interval/tolerance policy exists.
* If no quote snapshot exists within tolerance, record missing quote coverage explicitly.
* Do not fabricate best bid/ask from candle OHLC.
* Do not fail candle-only SMA research by default just because top-of-book is absent.

If changing manifest schema is too large, implement the internal loader and report fields first, and document that quote join is opt-in through a new explicit manifest field. Unsupported or unknown top-of-book fields must fail manifest parsing rather than being silently ignored.
