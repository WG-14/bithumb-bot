## Mission

Do **not** solve this by flattening the position, forcing resume, manually editing the DB, weakening safety gates, suppressing blockers, or repeatedly applying repair commands.

The goal is to make the `position_authority` repair/rebuild path strict about the correct invariant:

```text
final_safe_to_apply =
  broker/portfolio quantity convergence
  + provenance-aware semantic contract
  + no active fee/order/submit blockers
  + final post-assessment consistency
```

The goal is **not** to make the bot less strict. The goal is to make it strict about the right thing.

## Core design principle

`open_position_lots` must be treated as a rebuildable, provenance-aware materialized projection, not as untyped canonical truth.

Canonical truth should be:

```text
fills
trades
finalized fee events
broker fill evidence
external cash/position adjustments
manual repair events
current broker account snapshot as evidence
```

Derived projections should be:

```text
open_position_lots
trade_lifecycles
normalized exposure
dust tracking
operator recovery state
```

The repair/rebuild system must keep these meanings separate:

```text
FillNativeLot:
  source=ledger or accounted buy/fill evidence
  invariant=original fill qty + lifecycle residual

PortfolioAnchorProjectionLot:
  source=full_projection_rebuild_portfolio_anchor
  invariant=current broker/portfolio projection convergence
  may differ from original BUY fill qty

ManualRepairProjectionLot:
  source=portfolio_anchored_repair or equivalent
  invariant=repair event + broker/portfolio convergence
```

`target_lot_qty_fill_mismatch` may only be emitted for `FillNativeLot`.

  run the exact same postcondition assessment

This must apply to both:
