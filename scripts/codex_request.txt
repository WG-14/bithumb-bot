# Part 2: add attribution-based strategy sweep module

## Objective

Add the first `strategy_sweep` implementation that compares multiple `SmaStrategyConfig` combinations by running the existing candle-based replay and reading its `DecisionAttributionSummary`.

This is not a trading simulator. This is not a PnL backtest. This is not a fee-drag or max-drawdown report.

The sweep’s job is to answer:

```text id="iww58i"
For the same historical candles, how do different SMA/edge configs change:
raw BUY/SELL candidates,
final BUY/SELL candidates,
cost-filter blocking,
gap-vs-required-edge,
and primary attribution issue?
```
